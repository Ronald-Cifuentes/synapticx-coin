# ADR-0002: Base no PoS

## Contexto

El consenso de la capa base debe priorizar resistencia a captura política y oligárquica. PoS, dPoS y BFT con validadores visibles convierten capital previo y visibilidad institucional en poder de consenso.

## Decisión

La capa base **no** usará PoS, dPoS ni BFT con validadores visibles como mecanismo de soberanía. El consenso debe anclar el coste de ataque a un recurso externo (PoW), no a tenencia previa del activo.

## Estado epistemológico

[CONVICCIÓN ESTRUCTURAL] / [DESCARTADO] para PoS. Cerrado. No depende de investigación futura.

## Consecuencias

- PoW es la familia preferida por descarte
- IOTA Rebased, Penumbra y sistemas similares sirven como referencia de lo que no queremos en la base
- Cualquier propuesta de stake en la base se rechaza estructuralmente

## Qué lo invalidaría

Solo un cambio de objetivo: si el proyecto dejara de priorizar resistencia a captura sobre eficiencia o adopción institucional. No hay evidencia técnica que invalide el descarte; es una decisión de objetivos.
